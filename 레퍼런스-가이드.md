# Inklog 발표자료용 레퍼런스 가이드

교수님 피드백: "레퍼런스 사이트 캡처 이미지를 추가해서 발표 자료를 수정하면 좋겠다."
아래 사이트들을 캡처해서 발표 자료에 넣고, **각 캡처마다 "어떤 요소를 참고했고 Inklog에서 어떻게 변형했는지" 한 줄 코멘트**를 다는 것을 추천합니다.
단순히 "참고했다"가 아니라 "이 사이트의 A 요소를 → Inklog에서는 B로 적용했다" 형식이 설득력 있습니다.

---

## 1. 왓챠피디아 — pedia.watcha.com

**캡처할 화면**
- 영화 **작품 상세 페이지** (예: 아무 인기 영화나 검색)
  - 포스터 + 제목 + 감독 + 별점이 한 블록에 모여 있는 상단 영역
  - 하단의 "비슷한 작품" 추천 섹션

**발표 코멘트 예시**
> "왓챠피디아의 상세 페이지처럼 포스터·메타정보·별점을 한 블록에 배치했고,
> '비슷한 작품' 섹션은 Inklog에서 태그가 겹치는 기록을 보여주는 '함께 보면 좋은 기록'으로 변형했습니다."

**Inklog 연결점**: detail.html의 detail-hero 레이아웃, 관련 글 섹션

---

## 2. 키노라이츠 — kinolights.com

**캡처할 화면**
- **메인 페이지**: 큐레이션 섹션들이 세로로 이어지는 구조 (랭킹, 신작, 추천 등)

**발표 코멘트 예시**
> "키노라이츠 메인처럼 '최신 기록 → 베스트 → 통계' 순으로 섹션을 쌓아
> 빈 느낌이 들지 않는 메인 페이지를 구성했습니다."

**Inklog 연결점**: index.html의 섹션 구성 (통계 카운터, 최신 감상문, 베스트 기록, 월별 기록)

---

## 3. Goodreads — goodreads.com

**캡처할 화면**
- **My Books** 페이지 (읽은 책 목록 + 별점)
- **Year in Books** 또는 Reading Challenge (연간 독서 통계)

**발표 코멘트 예시**
> "Goodreads의 독서 통계처럼, 내가 남긴 기록을 숫자로 돌아볼 수 있도록
> 별점 분포 차트와 월별 기록 차트를 메인에 배치했습니다."

**Inklog 연결점**: 별점 분포 차트, 월별 기록 차트, 통계 카운터

---

## 4. 네이버웹툰 / 카카오페이지

**캡처할 화면**
- 웹툰 **작품 홈** (회차 목록 위 작품 정보 영역: 썸네일 + 제목 + 작가 + 별점/관심수)

**발표 코멘트 예시**
> "웹툰 플랫폼의 작품 정보 영역을 참고해, 웹툰 카테고리도 영화·책과 같은
> 카드 형식으로 통일감 있게 보여주도록 했습니다."

**Inklog 연결점**: 카테고리 공통 카드 디자인(entry-card), 갤러리 페이지

---

## 캡처 팁

1. **브라우저 전체화면(F11) 후 캡처** — 주소창·북마크바가 없는 깔끔한 화면
2. 캡처에 **빨간 박스로 참고한 영역 표시** → Inklog 스크린샷의 같은 영역에도 동일하게 표시하면 비교가 한눈에 들어옴
3. 슬라이드 구성 추천: 왼쪽 = 레퍼런스 캡처, 오른쪽 = Inklog 캡처, 아래 = 한 줄 코멘트
4. 출처 표기: 캡처 아래에 사이트명 + URL 작게 표기

## 발표에서 함께 언급하면 좋은 차별점

- 레퍼런스 사이트들은 모두 **서비스형(다른 사람의 평점 중심)**이지만, Inklog는 **나만의 기록 중심**
- 회원가입 없이 localStorage로 동작하는 완전 로컬 웹 → "개인 기록용 웹사이트"라는 교수님 코멘트와 연결
- JSON 내보내기/가져오기로 데이터 백업 가능 (localStorage 한계 보완)
